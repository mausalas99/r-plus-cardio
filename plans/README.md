**Cycle 2 (audit at `83049b1`, 2026-06-12):**

- `write-queue.js` "swallows errors": false — `enqueue` returns the un-caught `run` promise (callers awaiting it see rejections); the `.catch(() => {})` only keeps the serial chain alive. Standard pattern.
- 3s quit-flush window (`main.js` `Promise.race`): plan 003's own deliberate bounded-quit design, verified by its "app exits within ~3s" gate; persist failures are logged via `getLastPersistError`.
- Global (not per-IP) PIN lockout, in-memory only: plan 004 decided exactly this, with rationale (one ward per host; per-IP invites spoofing) and tuning knobs.
- `clinico-access-scope/` "zero test coverage": false — 40 plan-006 characterization goldens exercise `evaluateClinicalScope` through `clinico-access.test.mjs`. Per-evaluator unit tests would add marginal value; below the bar.
- Remove `R_PLUS_LAN_PERSIST_MODE` + legacy persistence paths: the rollback knob shipped 2026-06-08 (7.2.5), four days before this audit. Far too early; revisit after several stable releases on sql-v3.
- `ensureRoomBundle` missing `entityVersions.clinicalOps` init: single-threaded, first `putRoomClinicalOps` call sets it; no interleave point. Not worth a change.
- Partial cultivo date year inference (`completePartialFechaForCultivo`): speculative; invalid sets filtered upstream. Monitor.
- `labs.js` parsing efficiency (regex recompilation, chained replaces): real smell, but profile first — gated on the v8 Phase 0 profiling session above. No blind refactors of Tier-2 legacy.
- `panel.mjs` (3,160 lines) decomposition: no characterization suite exists; sequencing it before tests would repeat the mistake plan 008 fixed for orchestrator. Replay plan 019's playbook after characterizing it.
- `storage.js` cache-layer extraction: the parsed-blob cache invalidation is correct and tested; extraction is aesthetics this cycle.
- LAN characterization tests "lack edge cases": plan 008 scoped them as characterization deliberately; integration/race tests are a different (future) effort.
- `build:ui` watch mode: nice-to-have, below the leverage bar.
- `scripts/release.js` decomposition: HIGH risk on the release-critical path, low current pain.
- Repo-wide Tier-1 lint gate in CI: impossible today (2,242 pre-existing problems in `public/js`+`lib`+`lan-squad`); the changed-files ratchet inside `metrics:check` is the enforcement mechanism. Plan 012 scopes lint to root+scripts where green is achievable.
